METHOD 1: ONLINE (EASIEST - 10 MINUTES)
========================================

1. Go to: https://tinypng.com

2. Drag and drop images from:
   public\images\

3. TinyPNG will compress them automatically
   (Usually 60-70% size reduction)

4. Download compressed images

5. Replace originals in public\images\

PROS: Easy, no installation needed
CONS: Manual process, 20 images at a time

========================================
METHOD 2: SQUOOSH (GOOGLE'S TOOL)
========================================

1. Go to: https://squoosh.app

2. Upload each image

3. Adjust quality slider to 80%

4. Download optimized version

5. Replace original

PROS: More control, see before/after
CONS: One image at a time

========================================
METHOD 3: AUTOMATED (BEST FOR 119 FILES)
========================================

Step 1: Install Sharp
---------------------
npm install -g sharp-cli

Step 2: Backup Images
---------------------
xcopy /E /I public\images public\images-backup

Step 3: Compress PNG
---------------------
cd public\images
sharp -i *.png -o . --quality 80

Step 4: Compress JPG
---------------------
sharp -i *.jpg -o . --quality 80
sharp -i *.jpeg -o . --quality 80

Step 5: Compress WebP
---------------------
sharp -i *.webp -o . --quality 80

Step 6: Test Website
---------------------
cd ..\..
npm start

Visit: http://localhost:3001
Check if images look good

PROS: Fast, processes all images
CONS: Requires npm installation

========================================
METHOD 4: CONVERT TO WEBP (BEST COMPRESSION)
========================================

WebP format is 25-35% smaller than PNG/JPG!

Using Sharp:
cd public\images
sharp -i *.png -o webp\ -f webp --quality 80
sharp -i *.jpg -o webp\ -f webp --quality 80

Then update your views to use .webp files

PROS: Best compression
CONS: Need to update file references

========================================
QUALITY SETTINGS GUIDE
========================================

Quality 100% = Original size (no compression)
Quality 90%  = Barely noticeable, 30-40% smaller
Quality 80%  = Good quality, 50-60% smaller ✅ RECOMMENDED
Quality 70%  = Noticeable, 60-70% smaller
Quality 60%  = Poor quality, not recommended

========================================
WHICH IMAGES TO PRIORITIZE?
========================================

1. PNG files (usually largest)
   - Screenshots
   - Graphics with transparency
   - Logos

2. JPG files (photos)
   - Banner images
   - Product photos
   - Background images

3. WebP files (already compressed)
   - Can still optimize further

4. SVG files (vector)
   - Already small
   - Can minify with SVGO
